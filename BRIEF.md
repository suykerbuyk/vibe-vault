# LinkedIn Post — vibe-vault (Short Version)

---

**"Vibe coding" failed me. So I went back to what actually works.**

I pair-programmed with AI to build a 95,000-line industrial CAN bus control system in Rust — safety-critical hardware, 700+ tests, production systems software. "Vibe coding" was a complete disaster for it. What worked was the same paired programming discipline I'd practiced as an Agile evangelist for over a decade: treat the AI like a brilliant junior engineer with clear architectural guardrails, not a wish-granting machine.

But every session, the AI forgot everything. Architectural decisions, trade-offs, patterns, root causes — all evaporated into opaque transcript files.

So I built **vibe-vault** — an open-source Go binary that runs as a Claude Code hook to automatically capture every AI coding session into structured, cross-linked Obsidian notes. It extracts narratives, dialogue, git commits, and friction patterns. It scores related sessions across your project history. It scaffolds portable AI context that survives machine migrations and gives any new session instant recall.

Inspired by Dru Knox's "Context as Code" thesis — that AI context deserves the same engineering discipline as production code — vibe-vault grew from a session capture tool into infrastructure for project evolution tracking, portable AI memory, and behavioral observability.

428 tests. 44 development iterations. 10 completed phases. All built through the same AI pair-programming workflow it's designed to support.

The lesson: context is the bottleneck. Not the model. Not the prompt. The developers who thrive won't be the ones who prompt best — they'll be the ones who engineer the best context.

Open source (Apache 2.0 / MIT): https://github.com/suykerbuyk/vibe-vault
